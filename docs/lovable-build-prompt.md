You are helping build a production-grade JD Generator web app.

Build a full-stack SaaS web app with the following requirements.

## Product
An evidence-based Job Description Generator that integrates standard job/skill sources.

## Goal
A user can define a company role, match it against NCS, ESCO, O*NET, DigComp, and optionally SFIA, then generate a versioned JD draft with evidence trace, KPI suggestions, and dashboard management.

## Core Workflow
1. Source Hub
- Ingest and normalize NCS, ESCO, O*NET in v1
- Prepare DigComp for v1.1
- Keep SFIA behind feature flag until license is confirmed

2. Company Role Intake
- Collect title, org, mission, deliverables, recurring tasks, tools, stakeholders, and seniority

3. Matching Engine
- Show top candidate mappings with explainable scoring
- Include score breakdown and evidence preview
- Allow approve / reject / merge decisions

4. JD Generator
- Generate sections: role summary, mission, responsibilities, must-have, preferred, collaboration structure, core competencies, KPI suggestions, career path, evidence trace
- Every responsibility must reference evidence

5. Dashboard
- Role library by org tree
- Approval queue
- JD version history
- Evidence drawer
- Source sync status
- Duplicate job warnings
- Export to DOCX/PDF

## Tech Preferences
- Frontend: Next.js
- Backend: FastAPI or serverless API layer
- DB: PostgreSQL with pgvector
- Cache/queue: Redis
- Clean admin UI with enterprise dashboard pattern

## Data Model
Use the SQL schema in docs/jd_generator_v1_schema.sql as the base.

## Design Principles
- Enterprise, clean, minimal UI
- Strong table views and detail panes
- Left navigation, top summary cards, right evidence drawer
- Separate raw source content from generated JD content

## Non-Negotiables
- Explainable matching
- Evidence-linked JD generation
- Human review workflow
- Version control for JD records
- KPI suggestions generated by internal rules, not copied from external data
